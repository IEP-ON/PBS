## 2. IEP 영역 분담 구조

H.A.T.C.H.와 PBS 시스템이 IEP 전 영역을 분담한다.

| IEP 영역 | 담당 시스템 | 증빙 방법 |
|---|---|---|
| **교육과정 (교과 학습)** | H.A.T.C.H. 프로젝트 | 차시별 수행 기록·관찰 일지 포트폴리오 |
| **행동중재** | PBS 토큰 이코노미 | 주간 통장 거래내역·행동 빈도 그래프 |
| **일상생활 자립** | PBS 토큰 이코노미 | 출석·역할 기본급·루틴 달성률 |
| **통합교육 적응** | H.A.T.C.H. + PBS | H.A.T.C.H.: 발표회·통합학급 연계 / PBS: 선물 기능 |
| **의사소통** | H.A.T.C.H. + PBS | H.A.T.C.H.: 관찰 발표·기록 / PBS: 대체 표현 성과급 |
| **정서·사회성** | H.A.T.C.H. + PBS | H.A.T.C.H.: 생명 감수성 / PBS: 감정 표현·또래 활동 |

## 3. 학생별 PBS 목표 행동 × H.A.T.C.H. 연계

6명 학생의 PBS 목표 행동은 H.A.T.C.H. 수업 맥락에서 자연스럽게 발현된다.

| 학생 | PBS 핵심 목표 | H.A.T.C.H. 수업 연계 장면 |
|---|---|---|
| **서재민** | 물건 챙기기, 침·청결 자기인식, 사과 표현 | 돌봄 루틴 체크리스트 완료 → PBS 성과급 |
| **박창율** | 과제 지속, 발표 목소리, 감각 자기조절 | 관찰 일지 완성 → PBS 체크 |
| **김효주** | 대체 의사표현, 루틴 변화 순응 ⚠️ | 그림카드로 관찰 결과 표현 → PBS 성과급 |
| **조사영** | 도움 요청, 발표 시도, 스킨십 경계 | 소그룹 발표 시도 → PBS 체크 |
| **신지민** | 정서조절(울음 대체), 또래 관계 | 짝(박창율)에게 관찰 결과 말하기 → PBS |
| **민규원** | 공감 표현, 수업 복귀, 자기조절 | 또래 튜터 역할 수행 → PBS 성과급 |

## 4. Supabase 스키마 격리 원칙

두 시스템이 동일 Supabase 프로젝트를 공유하지만 **테이블은 완전 분리**한다.

```
PBS 토큰 이코노미 (이 앱) 테이블:
  class_codes, students, accounts, transactions,
  pbs_goals, pbs_records, salary_rules,
  shop_items, stock_prices, custom_stocks, custom_stock_prices,
  stock_holdings, behavior_contracts, contract_versions,
  behavior_functions, intervention_library, fba_records,
  ai_recommendations, extinction_alerts,
  class_account, class_transactions, dro_timers,
  consent_templates, system_settings

H.A.T.C.H. 인큐베이터 대시보드 테이블:
  incubator_readings, hatch_events, hatch_students,
  (별도 prefix 또는 별도 schema 권장)
```

**학생 ID 공유 여부:** 현재는 각각 독립 관리. 향후 통합 시 `students.name` 기준 매핑 가능.

## 5. 실제 수업 운영 흐름 (예시 — 5월 13일 차시 17)

```
수업 전
  PBS 시스템: 09:00 출석 기본급 자동 입금 ✅

수업 중 (40분 — "살아 있다는 뭘까?")
  H.A.T.C.H.: 병아리 분류 활동, C1 탐구 질문 답변 수집
  PBS 시스템: 교사가 수업 중 PBS 체크 (발표 시도, 자리 지키기 등)

수업 후
  PBS 시스템: 15:00 PBS 성과급 자동 정산
  → 서재민 발표 리더 역할: +200원
  → 신지민 짝 설명하기 성공: +150원

금요일
  PBS 시스템: 주간 보너스 + 이자 자동 정산
  → 통장 스냅샷 생성 (교사 1클릭 출력)
  → 통장 = IEP 성장 포트폴리오의 핵심 증빙
```

## 7. 개발 시 고려사항

PBS 토큰 이코노미 앱 개발 시 H.A.T.C.H. 관련해서 추가 구현이 필요한 사항:

### 현재 독립 구현 (이미 완료)
- PBS 체크, 토큰 지급, 통장 출력 — H.A.T.C.H.와 무관하게 작동

### 향후 연동 가능 (옵션)
- H.A.T.C.H. 수업 날짜 연계 → 특정 차시에 맞는 PBS 목표 자동 하이라이트
- 인큐베이터 온도 이상 감지 → PBS 시스템 알림 연계
- HATCH 발표회 날짜 → 커스텀 주식 "발표회 주식" 이벤트 자동 연계

### 데이터 중복 방지
- `students` 테이블은 PBS 시스템이 단독 관리
- H.A.T.C.H. 인큐베이터 대시보드가 학생 데이터 필요 시 PBS DB에서 읽기 전용 접근

## 8. 연구대회 보고서와의 연결

이 시스템은 수업연구대회 보고서 내에서 다음과 같이 포지셔닝된다:

```
보고서 내 위치: Part 1-6 "학급 행동지원 체계 — PBS 토큰 이코노미 시스템"

포지셔닝 원칙:
  · 이 시스템은 본 연구의 독립변수가 아니라,
    학급 일상의 구조화된 행동지원 체계로 기능한다.
  · H.A.T.C.H. 프로젝트 수업 중 학생들의 안정적 참여를 지원하는
    배경적 역할을 하되,
  · IEP의 교육과정 외 전 영역을 추적·강화하는 핵심 도구로서
    IEP 성장 심사와 직결된다.

보고서 심사 포인트:
  · AI·IoT 환경 구현 (라즈베리파이 + Claude Vision + 이 앱)
  · 교사 자체 개발 디지털 인프라
  · IEP 전 영역 추적 가능한 데이터 시스템
  · 통장 = IEP 성장 포트폴리오 증빙
```
