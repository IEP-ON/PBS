## 1. 프로젝트 개요

### 1.1 한 줄 정의
특수교육 대상학생의 PBS(긍정적 행동지원) 중재를 토큰경제(화폐·금융 시스템)로 구현한 AI 기반 행동지원 플랫폼

### 1.2 핵심 정체성
> 이 시스템은 단순한 용돈 앱이 아니라  
> **AI 기반 긍정적 행동지원 실행 플랫폼**이다.

- 화폐 = 토큰
- 가게 = 백업 강화제(backup reinforcer) 교환소
- 급여 = 토큰 획득 절차
- 통장 = 행동주의 포트폴리오
- 정산 = 토큰 교환 절차
- 주식 = 시장경제 교육 + 교사의 잔액 조정 도구

### 1.3 적용 대상
- 특수학급 (6~7명 소규모)
- 초등학교 기준 (파일럿: 남대구초등학교 특수학급)
- 확장 가능: 중·고등학교 특수학급, 타 학교 특수교사

### 1.4 기술 스택
- Frontend: Next.js
- Backend/DB: Supabase (PostgreSQL)
- 배포: Vercel
- AI: GPT API (OpenAI)
- 날씨: 외부 날씨 API (대구 → 학교 위치 기준)

### 1.5 배포 정책
- 무료 배포 (학운위 심의 무시)
- 로그인 회원제 아님 → **학급 식별코드 방식**
- 당장 IEP ON 흡수 없음, 동일 스택 유지

## 2. 접근 방식 설계 (인증 구조)

### 2.1 학급 개설 흐름
```
교사가 학급 개설 신청
    ↓
시스템이 학급 식별코드 자동 발급 (예: NDG-2026-001)
    ↓
해당 코드로 학급 전용 공간 생성
    ↓
교사가 학생 등록 (이름 + PIN 4자리 + QR코드 생성)
```

### 2.2 디바이스별 접근 방식

| 디바이스 | 접근 방법 | 비고 |
|---------|---------|------|
| 교사 PC | 학급 식별코드 + 교사 PIN | 관리자 권한 |
| 학생 태블릿 | 학급코드 → 학생 이름 선택 → PIN 4자리 | 간단하게 |
| 공용 ATM | QR카드 스캔 → 즉시 접근 | 빠른 인증 |

### 2.3 학부모
- **완전 배제** (통장 열람, 알림 등 일체 없음)
- 종이 통장 출력물로만 소통

### 2.4 QR ID 카드 역할
- 학생 신원 = 계좌번호
- ATM 빠른 접근 (PIN 입력 없이)
- 가게 구매 시 카드 제시

## 4. 운영 규칙 (확정)

### 4.1 화폐 기본값

| 항목 | 기본값 | 조정 주체 |
|------|-------|---------|
| 화폐 단위 | 500원 | 교사 (학급별 자유 설정) |
| 학기 초 시작 잔액 | 1,000원 | 교사 |
| 최저잔액 보호 | 500원 (반응대가 차감 불가 하한) | 시스템 고정 |
| 잔액 학기 이월 | 교사 결정 | 교사 |

### 4.2 가게 물가
- 교사가 아이템 등록 시 직접 설정
- 물가 조정 권한: 교사 전담
- 인플레이션 방지: 교사가 주기적으로 물가 점검

### 4.3 데이터 보관
- 학생 행동·거래 데이터: IEP 수립 기간과 동일하게 보관
- 보관 기간 만료 시 자동 삭제 또는 교사 수동 삭제
- 보호자 동의서: 관리 페이지 내 템플릿 제공

## 5. 급여 시스템

### 5.1 급여 원천 3종

| 분류 | 항목 | 기본값 | 정산 주기 |
|------|------|-------|---------|
| 기본급 | 출석 | 교사 설정 | 매일 자동 |
| 기본급 | 학급 역할 수행 | 교사 설정 | 매일 자동 |
| 성과급 | PBS 개별 목표 행동 | 학생별 상이 | 교사 체크 → 일 자동정산 |
| 보너스 | 주간 개근 | 교사 설정 | 금요일 자동 |
| 보너스 | 주간 PBS 목표 달성 | 학생별 상이 | 금요일 자동 |

### 5.2 자동 정산 스케줄
- 매일 오전 09:00 → 출석 기본급 입금
- 매일 오후 03:00 → 역할수행 + PBS 행동 체크분 일괄 입금
- 금요일 오후 02:00 → 주간 보너스 + DRL 달성 + 이자 계산
- 금요일 오후 02:30 → 주간 통장 자동 생성

### 5.3 PBS 연동 핵심 원칙
- 학생마다 개별 목표 행동 등록 (IEP 연동)
- 같은 행동이라도 학생마다 단가 다르게 설정 가능
- 교사는 수업 중 체크만 → 정산은 시스템 자동 처리

## 6. 주식 시스템 (확정 — 이중 구조)

### 6.1 구조 개요
```
날씨 연동 종목 (자동)     +     교사 커스텀 종목 (수동)
──────────────────────────────────────────────────
날씨 API 자동 갱신              교사가 언제든 주가 직접 조정
시장 원리 교육용                잔액 불균형 조정 도구
학생이 예측 가능                학생은 원리 모름 (보이지 않는 손)
```

### 6.2 날씨 연동 종목

| 종목 | 상승 조건 | 하락 조건 | 교육 개념 |
|-----|---------|---------|---------|
| ☀️ 아이스크림 주식 | 기온 25°C 이상 | 기온 10°C 이하 | 날씨-수요 관계 |
| 🍵 핫초코 주식 | 기온 10°C 이하 | 기온 25°C 이상 | 역상관 관계 |
| ☔ 우산 주식 | 강수 있는 날 | 맑은 날 | 날씨 예측·투자 |
| 🌸 나들이 주식 | 15~22°C 쾌적 | 극한 기온 | 적정 범위 개념 |

### 6.3 교사 커스텀 종목 (신규)

| 활용 목적 | 예시 종목 | 교사 조작 |
|---------|---------|---------|
| 잔액 하위 학생 지원 | 🎲 랜덤박스 주식 | 특정 학생 보유량 많을 때 폭등 |
| 학급 이벤트 연계 | 🎒 소풍 주식 | 소풍 전날 급등 |
| 수업 재료 | 📉 위기 주식 | 폭락 시나리오 시연 |
| 학생 참여 | 이름은 학생이 직접 짓기 | 교사가 관리 |

**커스텀 종목 운영 원칙:**
- 종목 이름: 학생들과 함께 정하기 (소유감 + 수업 재료)
- 주가 조정: 교사 PC에서 직접 숫자 입력 또는 폭등/폭락/보합 버튼
- 학생은 조작 여부 모름 → 자연스러운 시장 원리처럼 경험

### 6.4 이자
- 잔액 기준 이상 유지 시 주 단위 자동 이자 입금
- 저축 개념 체험용

## 7. 가게 시스템

### 7.1 아이템 분류

| 분류 | 예시 | 특이사항 |
|------|------|---------|
| 여가·활동권 | 자유시간 10분권, 게임시간권, 자리 선택권 | 소모성 쿠폰, 재고 무제한 |
| 실물 간식 | 사탕, 과자 | 재고 수량 관리 |
| 실물 물품 | 학용품, 스티커 | 재고 수량 관리 |
| 선물하기 | 위 아이템을 친구에게 | 사회적 행동 강화 |

### 7.2 선물 기능
```
A → B 선물 시
A 통장: -300원 "이○○에게 선물 — 자유시간권"
B 통장: +300원 "김○○에게 받은 선물 — 자유시간권"
→ 통장 = 사회적 관계 포트폴리오
```

### 8.2 출력 시점
- 매주 금요일 자동 생성 (1클릭 출력)
- 학기말 종합본 → IEP 보고서 첨부용

## 9. 행동계약서 시스템

### 9.1 근로계약서 형식 차용
학생이 계약의 무게감을 느끼고 주체적으로 참여하도록 설계

### 9.2 케이스별 계약서 유형 (샘플 문서 참조)
- Case A: 관심 기능 → DRA 전략 계약
- Case B: 회피 기능 → FCT + DRO 타이머 계약
- Case C: 행동형성 → 레벨업 단계 계약

### 9.3 GPT 자동 초안 생성
FBA 분석 결과 → 계약서 초안 자동 생성 → 교사 검토·수정 → 서명 → 시스템 자동 연동

## 10. AI 기능

### 10.1 FBA 분석 어시스턴트
행동 기술(자연어) + 선행사건 + 후속결과 + 빈도 데이터
→ GPT → 행동 기능 추정 + 중재 전략 제안 + 계약서 초안

### 10.2 소거 모니터링
강화 중단 감지 → 소거 폭발 위험도 분석 → 교사 경고 + 대응 가이드

### 10.3 AI 원칙
```
GPT는 "제안"만 한다.
최종 판단은 반드시 교사가 한다.
모든 화면에 고정 경고문 표시.
```
