# 프론트엔드 디자인 가이드
> 특수학급 PBS 토큰 이코노미 시스템
> 핵심 원칙: 지적장애·자폐성장애 학생이 혼자 쓸 수 있는 UI
> 최종 정리: 2026-03-21 (v4 — 두 버전 병합)

---

## 0. 사용자별 설계 전제

| 사용자 | 디바이스 | 설계 철학 |
|---|---|---|
| 학생 | 태블릿 (세로 768~1024px) | 접근성 최우선. 혼자 가능해야. |
| 교사 | PC (1280px 이상) | 기능 중심. 수업 중 한 손 조작. |
| ATM | 공용 태블릿 (가로 가능) | 3단계 이내. 실수 복구 가능. |

### 학생 사용자 특성 (설계 전제)

| 특성 | 설계 영향 |
|---|---|
| 읽기 어려움 (문자 인식 제한) | 이모지 + 짧은 단어 (3글자 권장) |
| 숫자 읽기 불안정 | 숫자 64px + 동전 시각화 병행 |
| 주의집중 짧음 | 화면당 핵심 정보 1~2개만 |
| 충동성 (버튼 막 누름) | 돈 나가는 행동 전 2단계 확인 필수 |
| 소근육·터치 정확도 낮음 | 버튼 최소 56px (권장 80px) |
| 흥미 유발 필요 | 애니메이션·이모지·밝은 색상 |

## 2. 학생 화면 레이아웃 상세

### 2.1 홈 (잔액)

```
┌──────────────────────────────┐
│  👋 민준아, 안녕!             │  ← 이름만, 28px Bold
│                              │
│      💰 내 잔액               │
│      2,450원                 │  ← 64px Bold
│      오늘 +300원 받았어요 ✅  │  ← 18px 초록색
│                              │
│  🪙🪙🪙🪙  + 450원          │  ← 동전(500원) 4개 + 나머지
│                              │
├──────────────────────────────┤
│  🛒     📊     📒     ✅     │  ← 하단 탭 4개
│  가게   주식  통장  체크     │  ← 80px 터치 영역
└──────────────────────────────┘
```

**동전 시각화 계산:**
```typescript
const coins = Math.floor(balance / 500)           // 🪙 개수
const remainder = balance % 500                   // 나머지 원
const maxDisplay = 8                              // 화면 최대 8개
const displayCoins = Math.min(coins, maxDisplay)
const hasMore = coins > maxDisplay               // 초과 시 "..." 표시
```

### 2.2 입금 알림 — 전체화면 오버레이 (필수 구현)

입금 발생 시 현재 화면 위에 4초간 표시.

```
┌──────────────────────────────┐  ← 배경: --color-success 전체
│                              │
│     💰💰💰💰💰               │  ← 동전 위에서 떨어지는 애니메이션
│   💰💰💰💰💰💰💰             │    (CSS keyframes, 1.5초)
│                              │
│     +200원 받았어요!          │  ← 28px White Bold
│   🎯 손 들고 발표하기         │  ← 이유 표시 (20px)
│                              │
│    잔액: 2,650원 💰          │  ← 새 잔액
│                              │
│    ┌──────────────┐          │
│    │    확인 ✓    │          │  ← 탭하면 즉시 닫힘
│    └──────────────┘          │
└──────────────────────────────┘

애니메이션 스펙:
- 진입: 0.3초 fade-in + scale(0.9→1)
- 동전: top:-100px → bottom 방향 translateY, ease-out
- 자동 닫힘: 4초 후 (setTimeout)
- 효과음: 없음 (교실 환경)
```

## 3. 교사 PC 화면 설계

교사 화면은 일반 웹앱 수준이지만, PBS 체크 화면은 별도 기준 적용.

### 3.1 PBS 체크 화면 (수업 중 한 손 조작 기준)

```
┌─────────────────────────────────────────────────┐
│ PBS 체크                   [오늘 정산 미리보기]   │
│ [민준] [서연] [태호] [지민] [규원] [효주]         │  ← 학생 탭
├─────────────────────────────────────────────────┤
│ 이민준                                           │
│                                                 │
│ ✅ 손 들고 발표   [1][2][3][4][5]  촉구: [□] |
│ □  자리 지키기   ⏱️ 12:34  [시작][리셋]         │
│ □  도움 요청     [✓ 체크]                       │
│                                                 │
│ 메모: [___________________________]             │
└─────────────────────────────────────────────────┘
```

- 클릭 최소화: 학생 탭 → 즉시 목표 목록
- 횟수 입력: 터치 카운터 [+1] 방식 또는 숫자 직접 입력
- DRO 타이머: 항상 상단에 크게 표시
- 체크 후 즉각 시각적 피드백 (초록 체크마크)

### 3.2 커스텀 주가 조정 UI

```
현재가: 100원

[직접 입력] 원하는 금액: [    원]
[🚀 폭등]  현재가 ×2~3배    → 예상: ~200~300원
[📈 상승]  현재가 ×1.2~1.5배 → 예상: ~120~150원
[➡️ 보합]  현재가 유지       → 예상: 100원
[📉 하락]  현재가 ×0.7~0.8배 → 예상: ~70~80원
[💥 폭락]  현재가 ×0.3~0.5배 → 예상: ~30~50원

내부 메모 (학생에게 비공개): [________________________]
```

## 6. Windsurf 구현 지침

### 반드시 지켜야 할 것 (비타협 사항)

```
① 학생 화면 모든 버튼 최소 56px (권장 80px) — 절대 작게 만들지 말 것
② 돈 나가는 모든 행동 → 2단계 확인 필수 (예외 없음)
③ 입금 발생 → 전체화면 초록 애니메이션 (4초, DepositAnimation 컴포넌트)
④ 잔액 = 64px 숫자 + 동전 시각화 항상 병행
⑤ 모든 버튼 아이콘 + 텍스트 함께 표시 (아이콘 단독 금지)
⑥ 학생 화면 텍스트 = 단어 수준만 (문장 사용 최소화)
⑦ 교사 커스텀 주식 teacher_note → 학생 화면에 절대 노출 금지
⑧ 빨간색 버튼 사용 금지 — 위험 상황은 텍스트+아이콘으로만
```

### 컴포넌트 구현 우선순위

```
Phase 1 필수 구현:
- BalanceCard       (잔액 + 동전 시각화)
- BigButton         (56px+ 대형 버튼)
- DepositAnimation  (전체화면 입금 피드백)
- ConfirmModal      (2단계 확인)
- ShopGrid          (이모지 아이템 카드)
- PbsCheckBoard     (교사 핵심 화면)

Phase 2:
- StockCard         (날씨·커스텀 주가 카드)
- TransactionList   (거래내역 날짜 그룹)
- DroTimer          (DRO 타이머 위젯)
- BankbookPrint     (통장 출력 레이아웃)
- QrScanner         (카메라 QR 인식)
```

### 사용 금지 패턴 (학생 화면)

```
❌ 드롭다운 메뉴
❌ 스와이프 제스처 (인식 어려움)

❌ 색상만으로 상태 구분 (아이콘 필수 병행)
```

## 8. 접근성 체크리스트

```
□ 모든 이미지 alt 속성
□ 버튼 aria-label 명시
□ 폼 입력 label 연결
□ 포커스 시 눈에 보이는 outline (2px solid Primary)
□ 색상만으로 정보 전달 금지 (아이콘 또는 텍스트 병행)
□ 터치 타겟 최소 56px
□ 에러 메시지 텍스트로 제공 (색상만 금지)
□ 로딩 상태 명시적 표시 ("잠깐만요" 텍스트 포함)
□ 중요 액션 전 확인 다이얼로그
□ 뒤로가기 버튼 항상 표시
```
