## 1. AI 기능 두 가지

### 기능 A: FBA 분석 어시스턴트
교사가 행동을 자연어로 설명 → GPT가 기능 분석 + 전략 제안

### 기능 B: 소거 모니터링 & 경고
이전에 강화되던 행동의 토큰 지급 중단 감지 → 소거 폭발 위험도 분석 + 교사 경고

### FBA 분석용 컨텍스트 빌더
```javascript
function buildFBAContext(student, recentRecords, antecedents, consequences) {
  return `
[학생 정보]
- 장애 유형: ${student.disability_type}
- PBS 지원 단계: ${student.pbs_stage}단계
- 기존 추정 기능: ${student.behavior_function || '미분류'}

[분석 요청 행동]
- 행동 기술: ${antecedents.behaviorDescription}
- 주요 선행사건 패턴: ${antecedents.patterns.join(', ')}
- 주요 후속결과 패턴: ${consequences.patterns.join(', ')}

[최근 4주 빈도 데이터]
${JSON.stringify(recentRecords, null, 2)}

위 데이터를 분석하여 행동 기능을 추정하고 
이 시스템(토큰경제)에서 적용 가능한 중재 전략을 제안해주세요.
  `;
}
```

### 소거 모니터링용 컨텍스트 빌더
```javascript
function buildExtinctionContext(student, goal, gapDays, recentHistory) {
  return `
[소거 감지 알림]
- 학생: ${student.name} (${student.disability_type})
- 목표 행동: ${goal.behavior_name}
- 추정 행동 기능: ${goal.behavior_function}
- 강화 미지급 경과일: ${gapDays}일
- 최근 30일 강화 이력: ${JSON.stringify(recentHistory)}

이 상황이 의도적 소거 절차인지 판단하고,
소거 폭발 위험도와 권장 대응 방안을 제시해주세요.
  `;
}
```

## 7. GPT 출력 형식 (표준)

```
【추정 행동 기능】
- 주요 기능: [기능명] (신뢰도: 높음/중간/낮음)
- 판단 근거: [데이터 패턴 기반 설명]
- 추가 관찰 필요 사항: [있으면 명시]

【추천 중재 전략】(우선순위 순, 최대 3가지)
1. [전략명 약어]: [구체적 실행 방법]
   → 토큰경제 연계: [이 시스템에서 적용 방법]
   → 근거: [출처]

2. ...

【주의사항】
- [윤리/안전 주의사항]

【소거 적용 시】(소거 논의 시에만 출력)
- 위험도: [높음/중간/낮음/적용불가]
- 병행 필수 전략: [명시]
- 안전 계획: [권장 사항]

⚠️ 이 제안은 보조적 도구입니다. 최종 판단은 담당 교사가 합니다.
```

## 8. AI 제안 피드백 루프

```
교사가 AI 제안 수락/거절
    ↓
ai_recommendations 테이블에 기록
    ↓
4주 후 효과 여부 교사 입력 (outcome_after_4weeks)
    ↓
[장기] 이 학교 현장 데이터 축적
    ↓
시스템 프롬프트 개선 또는 RAG 학습 데이터로 활용
```

## 9. 윤리 가드레일 7개 (프롬프트 적용)

1. AI 제안은 보조적 도구, 최종 판단은 교사
2. 근거 없는 제안 금지, 불확실 시 "추가 관찰 필요" 명시
3. 소거 단독 제안 금지 (DRA/FCT 항상 병행)
4. 반응대가: FBA + 보호자 동의 + 풍부한 강화 3조건 모두 충족 시만
5. 자동적 강화에 소거 절대 제안 금지
6. 자해/타해 위험 시 항상 전문가 팀 협의 권장
7. 학생 데이터 외부 공유 시 보호자 동의 안내
